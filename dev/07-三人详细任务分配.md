# 三人详细任务分配

这份文档是任务清单。任务不按“第几波”划分，而是按责任边界和依赖关系领取：先满足公共合同，再做自己的任务；依赖未完成时可以用已登记的 fake/fixture。

## 1. 领取任务前的共同准备

| 编号          | 任务                                        | 负责人     | 结果                                     |
| ----------- | ----------------------------------------- | ------- | -------------------------------------- |
| `ALL-00-00` | 冻结本地共同基线、确认 `LOCAL_ONLY=true`             | 用户      | 基线 commit、检查结果和本地任务目录                  |
| `ALL-00-01` | 只提取已有 fixture，登记来源和哈希                     | 全员，A 整理 | `tests/fixtures` 清单；不伪造黄金 INPUT/OUTPUT |
| `ALL-00-02` | 验证跨模块 fake codec/exporter、结果定位和 Editor 注册 | 全员，A 记录 | 跨模块合同测试通过                              |

`M0-GATE` 关闭前，可以修合同、整理 fixture、写 fake 和完成验证；正式功能任务开始前，三人要共同确认公共合同和任务边界。

## 2. A：工程、环境和海域

| 编号     | 任务                      | 主要内容                      | 交付/验收                     |
| ------ | ----------------------- | ------------------------- | ------------------------- |
| `A-01` | manifest 和类型映射          | 读取工程清单，识别节点类型和旧节点         | 清单可解析，未知类型保留              |
| `A-02` | `ProjectDocument` 和工程树  | 从清单构建工程对象、节点、ID 和资源引用     | 工程树与领域对象通过 ID 关联          |
| `A-03` | 打开工程 Service            | 隔离解压、调用 loader、保存源路径并报告错误 | 打开成功/失败都有 `CommandResult` |
| `A-04` | `EditorRegistry` 和主窗口接线 | 注册环境/海域及后续领域 Editor       | 未注册节点显示只读摘要               |
| `A-05` | 环境模型和校验                 | 环境参数、资源引用和字段校验            | 错误可定位到字段                  |
| `A-06` | 环境 codec                | 环境保存、读取和旧格式兼容             | 保存后可重新读取                  |
| `A-07` | 环境 INPUT exporter       | 生成 `Environment_in.dat`   | 与字段证据和 fixture 对比         |
| `A-08` | 海面和海底模型/Editor          | 表达海面、海底的最小需求              | 可被 B 使用并为未来三维提供对象         |
| `A-09` | 海域 codec 和 INPUT 导出     | 保存海域并生成对应 INPUT 片段        | 往返测试和文件对比通过               |
| `A-10` | 新建、保存、另存为、关闭            | 处理未保存修改和工程路径              | 保存后重新打开数据不丢失              |

## 3. B：浮体、水动力、工况和计算

| 编号     | 任务                            | 主要内容                       | 交付/验收                    |
| ------ | ----------------------------- | -------------------------- | ------------------------ |

## 7. 每张任务卡的最小内容

```text
目标 → 输入合同 → 修改文件 → 不做什么 → 测试/fixture
→ 验收证据 → 交给后续任务的接口 → agent 日志
```

## 8. 领取和交付规则

1. 先读本任务卡、`06` 合同和相关 fixture。
2. 依赖未完成时，登记 fake，不复制公共模型。
3. 一个任务对应一个本地变更和一份日志；运行测试、Ruff、compileall 并记录结果。
4. 发现公共合同、物理含义或黄金数据问题时暂停并交给用户决定。
